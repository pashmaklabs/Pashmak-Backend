برای اینکه بتونیم یه ورک فلو تمیز و سازماندهی شده داشته باشیم، یسری چیزا رو برا خودمون قانون گذاری میکنیم:
Branching -> ^(feature|bugfix|refactor|change)\/[a-zA-Z0-9_-]+$
رای هر issue یا فیچر جدید یه شاخه جدا درست کنین.
نحوه نامگذاری برنچامون اینجوریه: 
{branch_type}/{issue_title}
در کل چهار تا تایپ برنچو قرارداد میکنیم که استفاده کنیم:
feature : که برای دولوپ کردن یک فیچر جدید هست
bugfix : برای فیکس کردن باگ هایی که پیدا میکنیم
refactor : برای تغییر کد بدون تغییر فانکشنالیتی و عملکرد کد ( مثلا کلین کردن کد)
change : برای تغییر فانکشنالیتی کد ( مثلا تغییر فیچر های موجود)
اگر فیچر یا کاری بزرگ بود میتونید اونو بشکونید و برای هر کدوم یه برنچ بزنید.
Committing -> ^(feature|bugfix|refactor|change):\s.+\s#\d+$
هر کاری که می‌کنین، حتی اگر کوچیکه، یه کامیت جدید بزنین
فرمت نامگذاریش اینجوریه که اول نوع تغییر رو بنویسین، بعد توضیح بدین چیکار کردین(این توصح هم معمولا دستوری نوشته میشه و با فعل شروع میشه)، و آخرش شماره issue رو با # اضافه کنین. اینجوری اون عدد خودش لینک میشه به ایشو و نیاز نیست رفرنس بدین بهش.
{type}: {desc.} #{issue_id}
